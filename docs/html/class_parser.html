Detailed Description

Base class for performing an syntactic analysis over a set of tokens. A syntax tree can be produced if the methods are overriden correctly.

Constructor & Destructor Documentation

Parser::Parser ( )

Parser default constructor.

Constructs a Parser object without any tokenizer or LR table to use. These should be set before a call to the parse() method, as well as the end-of-input value to be used during parsing.

Parser::Parser ( ParserTable *  table,
Tokenizer *  tokenizer 
)

Parser constructor.

Constructs a Parser object with the given LR Table and Tokenizer objects. These two objects should be dynamically allocated, as this object will make sure they are properly deleted when this object is destroyed. The end-of-input value should still be set before calling parse().

Parameters
[in]tableThe LR Table to use.
[in]tokenizerThe Tokenizer to use.
Parser::~Parser ( )
virtual

Parser destructor.

Destroys a Parser object. Deallocates the table and tokenizer attributes, as well as emptying both the states and symbols stacks. If the syntax tree root was recovered, the user should make sure it is popped from the symbols stack or its value is set to 0, as the destructor will try to deallocate all SyntaxNode objects on the symbols stack.

bool Parser::parse ( const string &  line)
virtual

Parses the given line.

Parses a line, determining if it is properly written according to the syntax represented on the LR table. A tokenizer is used to obtain tokens from the line. If the proper methods are overriden properly, it will also create a SyntaxTree as a result of the parsing, and it will be available on the top of the symbols stack.

Parameters
[in]lineThe line to parse.
Returns
A boolean value indicating if the line was syntactically correct.
